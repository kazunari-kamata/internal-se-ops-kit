# アカウント管理

アカウント管理は、利用者が必要な権限で業務を始められることと、不要になった権限を確実に停止することを両立する運用です。申請、承認、設定、確認、記録の流れを分けて扱うと、作業の抜け漏れを減らせます。

## 基本方針

- アカウント作成、変更、削除は申請記録に紐づける
- 権限は業務に必要な範囲に限定する
- 共有アカウントは原則として作らない
- 例外対応は期限と責任者を記録する
- 退社、異動、休職などのイベントを起点に権限を見直す

## 入社時の流れ

1. 利用者ID、所属区分、開始日、必要な業務ロールを確認する
2. 承認者が必要な権限を確認する
3. アカウントを作成する
4. 初期設定と多要素認証の案内を行う
5. 利用者または依頼元に利用開始を確認する
6. 作業記録を保存する

## 変更時の流れ

権限追加や部署変更では、既存権限を残したまま追加するだけになりがちです。変更申請では、追加する権限と削除する権限を分けて確認します。

## 定期レビュー

アクセスレビューでは、利用者、権限、最終確認日、承認者、対応結果を記録します。不要な権限は発見した時点で削除予定日を決め、完了後に記録を更新します。

## 記録する項目

- 申請ID
- 利用者ID
- 対象システム
- 権限種別
- 承認者区分
- 作業日
- 作業者区分
- 確認結果

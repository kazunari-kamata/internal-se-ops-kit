# インシデント対応

インシデント対応では、初動の速さと記録の正確さが重要です。影響を抑える対応と、後から振り返れる記録を同時に進めます。

## 初動

1. 受付時刻と申告内容を記録する
2. 影響範囲を仮置きする
3. 緊急度を判定する
4. 必要に応じて一次対処を行う
5. 関係者へ状況を共有する

## 切り分け

確認済みの事実、未確認の仮説、実施した操作、変化した状況を分けて記録します。変更作業の直後であれば、変更内容と戻し方を確認します。

## 復旧

復旧作業では、作業者、開始時刻、終了時刻、実施内容、確認結果を残します。復旧後は、影響が解消したことを依頼元または確認者に確認します。

## 事後対応

再発防止策は、具体的な作業として書きます。例として、監視条件の追加、手順書の更新、権限レビュー、利用者向け案内の改善があります。期限と担当区分を決め、完了後に記録を閉じます。

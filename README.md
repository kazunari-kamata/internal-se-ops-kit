# internal-se-ops-kit

社内SE業務で発生しやすい運用作業を、テンプレート、チェックリスト、小さな自動化スクリプトとして整理するための公開用サンプルリポジトリです。

このリポジトリに含まれる内容はすべて架空のサンプルです。実在する会社名、顧客名、個人名、メールアドレス、IPアドレス、ドメイン、社内パス、ローカル環境固有の絶対パスは含めません。

## 内容

- `docs/`: 社内SE向けの運用説明
- `templates/`: 入退社、アクセスレビュー、インシデント、変更申請のテンプレート
- `scripts/`: サンプルデータ検査とチェックリスト生成の小さな Python スクリプト
- `examples/`: 架空のサンプルCSVとレポート
- `.github/`: GitHub Actions と PR テンプレート

## 想定用途

- 定型作業の抜け漏れ防止
- 運用手順の初期ドラフト作成
- 棚卸し用CSVの最小限の形式チェック
- 入社・退社対応のチェックリスト出力

実際の業務で使う場合は、自組織の規程、利用サービス、承認経路、監査要件に合わせて項目を見直してください。

## 必要な環境

- Python 3.12
- pytest

pytest が未導入の場合は、任意の仮想環境を有効化してから次を実行します。

```sh
python -m pip install pytest
```

## コマンド

サンプル資産CSVを検査します。

```sh
python scripts/validate_inventory.py examples/sample-assets.csv
```

入社チェックリストを出力します。

```sh
python scripts/generate_checklist.py onboarding
```

退社チェックリストを出力します。

```sh
python scripts/generate_checklist.py offboarding
```

テストを実行します。

```sh
python -m pytest
```

## 公開時の注意

- 実データをコミットしない
- 個人名やメールアドレスをテンプレートの例に使わない
- サンプルであっても実在するネットワーク情報を書かない
- 社内の共有フォルダ、端末名、チケットURL、監視画面URLを含めない
- コミットメッセージや PR 本文にも機密情報を含めない
